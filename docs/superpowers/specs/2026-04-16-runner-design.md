# Runner (도망자) — 설계 문서

**날짜:** 2026-04-16  
**프로젝트:** Runner — 도망자 보드게임 온라인 구현  
**스택:** React + TypeScript + Vite + Firebase Realtime Database + GitHub Pages

---

## 1. 개요

도망자 보드게임을 웹/모바일에서 1대1 온라인으로 플레이할 수 있는 앱.  
GitHub Pages에 정적 배포하며, Firebase Realtime Database로 실시간 게임 상태를 동기화한다.  
친구끼리만 플레이 — 방 코드를 공유해서 입장하는 방식.

---

## 2. 게임 규칙 (구현 기준)

### 카드 구성
- 전체 카드: 0~42
- 0: 시작점 (보드에 고정)
- 1, 2, 3, 42: 도망자 시작 손패
- 4~14 (11장), 15~29 (15장), 30~41 (12장): 3개 더미로 분리
- 추격자: 추리용 보드판 (1~42 숫자 그리드) 제공

### 도망자 규칙
- 카드를 **뒷면**으로 경로에 놓으며 진행
- 놓는 카드는 마지막 카드 숫자 + 1~3 범위여야 함
- 범위를 초과하려면 **부스터 카드**를 해당 카드 밑에 쌓음
  - 홀수 부스터: +1 발자국 (범위 1 확장)
  - 짝수 부스터: +2 발자국 (범위 2 확장)
  - 부스터 여러 장 누적 가능
- 턴마다 카드를 놓거나 **패스** 가능
- 42를 경로에 놓으면 **도망자 승리**

### 추격자 규칙
- 뒷면 카드 중 원하는 것을 선택해 숫자 추리
- 선택한 카드 **전부 맞아야** 뒤집힘 (하나라도 틀리면 아무것도 뒤집지 않음)
- 추리 성공 시 해당 카드 + 밑에 쌓인 부스터 카드까지 모두 공개
- 모든 뒷면 카드를 뒤집으면 **추격자 승리**

### 첫 턴 특수 규칙
| 역할 | 드로우 | 추가 행동 |
|------|--------|-----------|
| 도망자 | 4~14에서 3장 + 15~29에서 2장 (총 5장 추가) | 카드 최대 2장 뒷면으로 놓기 가능 |
| 추격자 | 원하는 더미에서 2장 | 이후 즉시 추리 가능 |

### 이후 턴 (번갈아 진행)
- 도망자: 원하는 더미에서 1장 드로우 → 카드 놓기 or 패스
- 추격자: 원하는 더미에서 1장 드로우 → 추리 (0장 이상 선택)

```

└── Vite + React + TypeScript 빌드 결과물
```

## 5. Firebase 데이터 구조

```
rooms/{roomId}/
  status: 'waiting' | 'playing' | 'finished'
  turn: 'runner' | 'chaser'
  phase: 'draw' | 'action'
  turnNumber: number          # 0부터 시작, 첫 턴 특수 규칙 분기에 사용

  players:
    runner: { name: string }
    chaser: { name: string }

  trail: TrailCard[]
    TrailCard:
      face: 'down' | 'revealed'
      value?: number           # 도망자만 알 수 있음 (face='down'이면 추격자에게 숨김)
      booster?: number         # 밑에 쌓인 부스터 카드 숫자 (있을 경우)

  piles:
    low:  number[]             # 4~14 남은 카드
    mid:  number[]             # 15~29 남은 카드
    high: number[]             # 30~41 남은 카드

  runnerHand:  number[]        # 도망자 손패 (도망자 클라이언트만 표시)
  chaserHand:  number[]        # 추격자가 뽑은 카드

  chaserBoard:                 # 추격자 추리 보드
    { [key: 1~42]: 'unknown' | 'eliminated' | 'correct' }

  guessAttempt: number[]       # 현재 턴 추리 선택 중인 카드 목록

  winner?: 'runner' | 'chaser'
```

> **보안:** `runnerHand`와 trail의 `value`는 Firebase Security Rules로 도망자 본인만 읽을 수 있도록 제한.

## 7. 카드 하이라이트 UX

도망자가 카드를 놓을 때 손패에서 플레이 가능한 카드를 시각적으로 강조:

```
기본 상태:
  마지막 경로 카드 = N
  → 손패 중 N+1 ~ N+3 범위 카드에 금색 테두리 하이라이트

부스터 카드 선택 시:
  홀수 부스터 선택 → 범위 +1 확장 → N+1 ~ N+4 하이라이트
  짝수 부스터 선택 → 범위 +2 확장 → N+1 ~ N+5 하이라이트
  부스터 여러 장 누적 시 발자국 수만큼 누적 확장
  → 실시간으로 하이라이트 업데이트
```
